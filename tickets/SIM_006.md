# SIM-006: The Plate Check (Behavior Logging)

## üéØ OBJECTIVE
Build the "Plate Check" UI to replace manual calorie logging. This interface allows users to log meals based on **Behaviors** (Protein/Plants) and **Satiety**, writing to the new `days/{date}` schema.

## üë§ USER STORY
**As a** User,
**I want** to log my meals in under 5 seconds by checking off simple habits,
**So that** I stay accountable without becoming an accountant.

## ‚úÖ ACCEPTANCE CRITERIA
1.  **UI - The Modal:**
    * **Trigger:** Tapping "Check In" on the Home Screen.
    * **Meal Selector:** Simple toggle (Breakfast / Lunch / Dinner / Snack).
    * **The Pillars (Binary Toggles):**
        * "Did you have palm-sized Protein?" (Yes/No)
        * "Did you have 1-2 fists of Plants?" (Yes/No)
    * **The Satiety Slider (Intuition Calibration):**
        * Range: 1 (Starving) to 5 (Stuffed).
        * Labels (Scientific/Witty): "Empty" ‚Üí "Neutral" ‚Üí "Satiated" ‚Üí "Thanksgiving".
2.  **Logic & Data:**
    * Save to `users/{uid}/days/{YYYY-MM-DD}` using `setDoc` (merge).
    * Field Structure: `meals: { breakfast: { protein: true, plants: false, satiety: 3 } }`.
3.  **Home Screen Update:**
    * The "Nourishment Card" must update to show *Habits Completed* (e.g., "2/3 Protein Goals Hit") instead of "Calories Remaining."
    * **Reward:** If the user hits "Protein = True", trigger the Holographic Shimmer.
